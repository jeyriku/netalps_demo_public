# NetAlps Public Demo

Public, anonymized copy of the original NetAlps demo.

This repository demonstrates a small network CI validation workflow with:
- Containerlab
- FRRouting (OSPF + BFD)
- PyATS / Genie tests

The topology uses only public container registries.

## Public Container Images

- `frrouting/frr:latest`
- `nicolaka/netshoot:latest`

## Project Layout

- `topology.clab.yml`: Containerlab topology (2 FRR routers + 2 hosts)
- `configs/`: FRR startup configs
- `tests/`: PyATS checks (`pre_check`, `post_check`, and job file)
- `scripts/failure_demo.sh`: intentional failure scenario with rollback

## Prerequisites

- Linux host
- Docker running
- Containerlab installed
- Python 3.10+ with a virtualenv
- PyATS + Genie installed in the venv

Example install:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install pyats genie
```

## Quick Start (Local)

```bash
# 1) Pull public images
docker pull frrouting/frr:latest
docker pull nicolaka/netshoot:latest

# 2) Deploy lab
containerlab deploy -t topology.clab.yml --reconfigure
sleep 20

# 3) Run validations
python tests/pre_check.py --testbed tests/testbed.yml
python tests/post_check.py --testbed tests/testbed.yml

# Optional: full PyATS job
pyats run job tests/test_job.py --testbed-file tests/testbed.yml

# 4) Destroy lab
containerlab destroy -t topology.clab.yml --cleanup
```

## Failure Demo

```bash
source .venv/bin/activate
bash scripts/failure_demo.sh
```

The script injects an OSPF mismatch on `frr-rtr-02`, confirms test failure,
then restores the original config and verifies recovery.

## GitHub Actions (Optional)

A workflow is provided in `.github/workflows/lab-validation.yml`.

It is designed for a `self-hosted` runner that already has:
- Docker
- Containerlab
- Python + dependencies

GitHub-hosted runners do not support this lab out of the box.
